--- layout: main title: skuid.ui --- skuid.ui Object ================= ## Classes ### Editor Class #### Constructor ##### `new skuid.ui.Editor( parentElement, options )` Instantiates a new editor. `parentElement` is the parent dom element that this editor will attach itself to. The `options.showSaveCancel` property will tell the editor whether or not to create a save and cancel button for this editor. #### Prototype Methods ##### `registerModel( model )` Indicates to a model that this editor will be editing it. Once registered, an editor will be notified when changes are made to the model. If a change is made, `handleChange()` will be called. If new data comes in from the server, `handleDataRefresh()` will be called. If the model is saved or cancelled, `handleSave()` or `handleCancel()` will be called. ##### `save()` Saves all models associated with this editor. ##### `cancel()` Cancels all models associated with this editor. #### Overriding and Extending Editors Skuid Editors have for prototype methods that can be overridden or extended. The four methods are `handleChange()`, `handleDataRefresh()`, `handleSave()` and `handleCancel()`. {% highlight javascript %} // Instantiate a new skuid.ui.Editor var myeditor = new skuid.ui.Editor(parentElement); // Override the default handleChange() functionality myeditor.handleChange = function() { // Handle the change } // Extend the default handleChange() functionality myeditor.handleChange = function() { skuid.ui.Editor.prototype.handleChange.call(myeditor); // Handle the change } {% endhighlight %} ### Field Class #### Constructor ##### `new skuid.ui.Field( row, model, editor, config )` Instantiates a new DOM element for displaying / interacting with a particular field on a particular row in a particular model. The SKuid JavaScript equivalent of the `` tag. Example: {% highlight javascript %} // Get a reference to the model and row we'll be working with var model = skuid.model.getModel('AccountData'), row = model.getFirstRow(); // Create a DOM element that will become our Skuid Editor var element = $('
'); // Create a new Skuid UI Editor, and register it on our model var editor = new skuid.ui.Editor(element); editor.registerModel(model); // Create a new skuid.ui.Field corresponding to the 'MailingCity' field var f = new skuid.ui.Field( row, model, editor, { fieldId: 'MailingCity', register: true } ); // Actually render the field, running the corresponding field renderer logic // using the metadata Skuid finds for the 'MailingCity' field. f.render(); {% endhighlight %} #### Prototype Methods ##### `render()` Actually renders the field, creating or modifying field.element, a DOM element. Runs the appropriate field renderer logic based on the specified data field's metadata. For instance, if field.id is 'MailingCity' on the Account object, the 'TEXT' renderer will be used. If field.id was 'CreatedDate', then the 'DATE' renderer will be used. #### Overriding Field prototype methods Skuid Fields have one prototype method that may need to be overriden: `handleChange()`. This will only be called by Skuid if the skuid.ui.Field is registered on a particular model. If a change is made to the skuid.ui.Field's corresponding row and field in that model, then Skuid will call the skuid.ui.Field's `handleChange()` method as soon as the model has finished processing the change. ## Properties / Methods #### Field Rendering ##### `fieldRenderers` An object containing all of Skuid's standard renderers for each of Salesforce's Display Types. The following Display Types are included in `skuid.ui.fieldRenderers`: - BOOLEAN - DATETIME - DATE - TEXT (handles URLs as well) - TEXTAREA - EMAIL - CURRENCY - PERCENT - DOUBLE - INTEGER - PICKLIST - MULTIPICKLIST - REFERENCE: for Lookup/Master-Detail fields - ID There are also a few other renderers that are used to handle special Skuid field rendering functionality: - CHILDREL: for iterating over Child Relationship data and processing a per-row Template - COMBO: for Template fields - CUSTOM: for running a custom field renderer (defined with a Snippet) - FILE: for rendering a lookup to a Skuid Image object as a File Uploader / display. - REFPICK: for rendering a reference field (lookup/master-detail) as a picklist Some of these field renderers include multiple render methods, which can be selected by setting `field.options.type` --- which corresponds to the "Field Renderer" property in the Skuid Page Builder. For instance, if the MULTIPICKLIST renderer is handed field.options.type = 'CHECKBOXES', then it will output a series of checkboxes for each Picklist Entry for that field. Each renderer, e.g. 'BOOLEAN', is actually an object with one to three properties, corresponding to different Render Modes: 'read','edit', 'readonly' (readonly is new in Summer 13). So to execute a renderer on a particular field with a particular value, do something like this: {% highlight javascript %} // Assuming that uiField is a skuid.ui.Field on a Date field, // such as the Contact Birthdate field, // this will run the DATE renderer logic on that Field. skuid.ui.fieldRenderers.DATE.edit(uiField,'2013-07-13'); {% endhighlight %}
